## 3. LLM 기반 키워드 추출 파이프라인

모든 LLM 추출은 **로컬에서 실행 중인 Ollama 서버** (`http://localhost:11434`)와 **gemma4:e4b 모델**을 사용한다. 외부 API 호출 없이 온프레미스로 처리하여 데이터 유출 위험 없이 운영된다.

### 공통 전처리 (`preprocess_instagram_text`)

인스타그램 게시글 특성상 아래 전처리를 LLM 입력 전에 공통 적용한다.

```
1. title + body 결합
2. @계정명 제거 (정규식: @\w+)
3. 해시태그 기호(#) 제거, 단어는 유지
4. 연속 공백·줄바꿈 정규화
5. 최대 500자 절단
```

---

### 3-1. POS 상품명 키워드 추출

| 항목 | 내용 |
|---|---|
| **함수** | `extract_keywords(text)` |
| **입력** | POS 상품명 문자열 |
| **프롬프트** | `SYSTEM_PROMPT` |
| **출력 형식** | `단어1, 단어2, ...` (comma-separated 리스트) |
| **추출 목표** | HIN 노드로 연결 가치가 높은 핵심 명사 5~10개 |

**프롬프트 핵심 로직**

- 상품명에서 [상품 속성], [TPO/소비 맥락], [마케팅/트렌드] 관련 키워드 식별
- 서술어·조사·일반 부사 소거
- 명사 및 명사구 5~10개 압축 출력

---

### 3-2. POS 상품 속성 추론

| 항목 | 내용 |
|---|---|
| **함수** | `infer_attributes(product_name)` |
| **파일** | `attribute_inferrer.py` |
| **입력** | POS 상품명 |
| **프롬프트** | `SYSTEM_PROMPT` (attribute_inferrer 전용) |
| **출력 형식** | `맛/온도/식감/TPO` 등 지정 어휘 쉼표 구분 리스트 |

**프롬프트 핵심 로직**

지정된 7개 속성 카테고리(맛, 온도, 식감, 취식 편의성, 건강·라이프스타일, 소비 목적/TPO, 용량·포장 형태) 내 단어만 출력하도록 엄격히 제한한다. 카테고리 어휘 외 출력·유의어 사용 금지, Few-Shot 예시 4개 포함.

```
Input: 코카콜라 제로 500ml
Output: 단맛,차가움,바로취식(RTE),제로슈거,제로칼로리,당충전,1인용(소포장)
```

### 3-3. knewnew 트렌드 게시글 키워드 추출

| 항목 | 내용 |
|---|---|
| **함수** | `extract_keywords_instagram(title, body)` |
| **배치 스크립트** | `batch_instagram_keywords.py` |
| **입력** | knewnew 인스타 게시글 `title` + `body` |
| **프롬프트** | `INSTAGRAM_SYSTEM_PROMPT` |
| **출력 형식** | `단어1, 단어2, ...` (comma-separated 리스트) |
| **처리 현황** | 1,037건 중 **1,035건 완료** |

**프롬프트 핵심 로직**

knewnew는 브랜드 계정이 아닌 트렌드 미디어이므로, 소비 트렌드 신호 포착에 집중한다.

- **지역명 조건부 판별**: 단순 방문 위치(`강남`, `홍대`)는 제거, 트렌드 수식어로 쓰인 지역명(`두바이 초콜릿`, `도쿠시마 라면`)은 유지
- **복합명사 분리**: `메이플 페스타` → `메이플, 페스타`
- **정규화**: `Zero sugar` → `제로슈거`, `pop-up` → `팝업`
- 추출 우선순위: ① 맛·식감·원재료 트렌드 → ② 트렌드 수식 지역명 → ③ TPO·소비 맥락 → ④ 유통채널·브랜드 → ⑤ 콜라보·IP

**체크포인트 방식**: 10건마다 중간 저장 → 중단 후 재시작 시 기처리 게시글 자동 스킵

### 3-4. 세븐일레븐 공식 인스타 구조화 추출

| 항목 | 내용 |
|---|---|
| **함수** | `extract_keywords_seveneleven(title, body)` |
| **입력** | 세븐일레븐 공식 인스타 게시글 |
| **프롬프트** | `SEVENELEVEN_SYSTEM_PROMPT` |
| **출력 형식** | 구조화 JSON |

**출력 JSON 스키마**

```json
{
  "metadata": [
    {"name": "상품명", "price": 2500, "capacity": "300ml 또는 null"}
  ],
  "flavor_and_category": ["말차", "쫀득", "머랭쿠키"],
  "collab_and_brand": ["미미미누", "헬로키티"],
  "promotion_type": ["2+1", "단독"],
  "tpo_context": ["야식", "당충전"]
}
```

**프롬프트 핵심 로직**

- 게시글 유형 판별 (신상 소개 / 이벤트·프로모션 / 콜라보 공지)
- 상품명·가격·용량 등 메타데이터 원문 그대로 추출 (추정 금지, 없으면 `null`)
- 세븐일레븐 자체 브랜드명(`세븐일레븐`, `7eleven`) 출력 제외
- 프로모션 표기 정규화: `원플원` → `1+1`, `투플러스원` → `2+1`

## 5. 파이프라인 흐름 요약

```
[수집]
  Apify Instagram Scraper
    ├── 편의점 공식 계정 (세븐일레븐, CU, GS25)
    └── 트렌드 미디어 (knewnew)

[전처리]
  preprocess_instagram_text()
    └── @계정명 제거 / 해시태그 기호 제거 / 공백 정규화 / 500자 절단

[LLM 추출] — Ollama (gemma4:e4b, 로컬)
  ├── POS 상품명     → extract_keywords()         → 키워드 리스트
  ├── POS 상품명     → infer_attributes()          → 속성 어휘 리스트
  ├── knewnew       → extract_keywords_instagram() → 트렌드 키워드 리스트
  ├── 세븐일레븐 공식 → extract_keywords_seveneleven() → 구조화 JSON
  └── CU 공식        → extract_keywords_cu()       → 구조화 JSON

[배치 처리]
  체크포인트 방식 (10건마다 저장) → 중단/재시작 안전
  결과 저장: data/processed/*.csv
```
