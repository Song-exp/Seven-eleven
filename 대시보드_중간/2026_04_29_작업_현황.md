## 프로젝트 개요

세븐일레븐 내부 상품 데이터 + 커뮤니티 트렌드 데이터를 결합한 **지식 그래프 기반 NPD 대시보드** 파이프라인.

- **목표:** 트렌드 키워드 입력 → Gemma LLM 속성 추론 → 네트워크 BFS → 신제품 속성 조합 제안
- **배포:** 대시보드(Vercel) + 백엔드(로컬 FastAPI + ngrok)
- **LLM:** `gemma4:e4b` via Ollama (로컬) / 외부 Gemma API (시연)

---

## 디렉토리 구조

```
대시보드_중간/
├── dashboard.html              ← 프론트엔드 (Vercel 배포 대상)
├── config.js                   ← API URL/Key 설정
├── server.py                   ← FastAPI 백엔드
├── network_builder.py          ← 그래프 로더 (이중 모드)
├── graph_engine.py             ← 가중치 BFS 탐색
├── llm_connector.py            ← Gemma 호출 모듈 (로컬/원격)
├── build_master.py             ← 원본 데이터 → parquet 마스터 생성
├── create_dummy_data.py        ← 테스트용 더미 network.json 생성
├── trend_cache_builder.py      ← 트렌드 속성 사전 캐싱
├── requirements.txt
├── .env.example                ← GitHub 공개용 환경변수 예시
├── .env                        ← 실제 환경변수 (gitignore됨)
├── .gitignore
└── data/
    ├── network.json            ← 현재: 더미 데이터 (실제 데이터 수령 후 교체)
    └── trend_cache.json        ← 트렌드 캐시 (gemma 추론 후 생성)
```

## 서버 시작 명령어

### 최초 1회 설정

```powershell
# 1. 패키지 설치
pip install -r requirements.txt

# 2. 환경변수 설정
copy .env.example .env
# .env 파일 열어서 값 수정

# 3. 더미 데이터 생성 (실제 network.json 없을 경우)
python create_dummy_data.py
```

### Ollama 실행 (로컬 LLM)

```powershell
# Ollama 백그라운드 실행 (최초 설치: https://ollama.com)
ollama run gemma4:e4b
# 테스트 후 /bye 입력 → Ollama는 백그라운드 유지됨
```

### FastAPI 서버 실행

```powershell
# 개발 모드 (파일 변경 시 자동 재시작)
uvicorn server:app --port 8000 --reload

# 운영 모드
uvicorn server:app --host 0.0.0.0 --port 8000
```

### ngrok 터널 (외부 공개 시)

```powershell
# 새 터미널에서 실행 (서버는 계속 켜둬야 함)
ngrok http 8000
# → 발급된 https://xxxx.ngrok-free.app 주소를 config.js의 API_BASE_URL에 입력
```

### E2E 테스트

```powershell
python e2e_test.py
```

## LLM 모드 전환 (.env만 수정)

```ini
# 로컬 테스트
LLM_MODE=local
OLLAMA_MODEL=gemma4:e4b

# 시연 — 외부 API 서버
LLM_MODE=remote
REMOTE_LLM_URL=https://xxxx.ngrok-free.app
REMOTE_LLM_KEY=YOUR_SECRET_KEY
```

> **.env 파일은 .gitignore에 포함 → GitHub에 올라가지 않습니다.**  
> 신규 환경 설정 시 `.env.example`을 복사하여 사용하세요.
